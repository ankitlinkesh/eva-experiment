# Eva Demo Commands

- `eva release status`: show the Phase 29 local profile.
- `eva release demo`: show the public walkthrough summary.
- `eva release commands`: show this report-only command family.
- `eva release capability map`: show available and locked surfaces.
- `eva release safety proof`: show deterministic safety evidence.
- `eva release readiness`: show local demo-review readiness.
- `eva release limitations`: show honest limitations and non-goals.
- `eva release verification`: show manual verifier commands.

Natural `eva ask` equivalents cover release status, public demo, commands, capability map, safety proof, readiness, limitations, and what Eva can or cannot do.

Every response states that no publishing, commit, secret access, provider call, browser/desktop control, source editing, shell/test/package/git execution, or unrestricted crawling occurred.

These commands do not execute verification; they only show the commands a developer should run.
