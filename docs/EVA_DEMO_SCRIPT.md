1. Run `eva release status` and explain that Phase 29 is local report/status only.
2. Run `eva release demo` for the bounded public overview.
3. Run `eva release capability map` to separate available previews from locked execution.
4. Run `eva release commands` to show the deterministic demo surface.
5. Run `eva release safety proof` and call out the unchanged execution gates.
6. Run `eva release limitations` before discussing readiness.
7. Run `eva release verification` to show the manual evidence bundle.
8. Run `eva release readiness` last; it must not be treated as publication.
